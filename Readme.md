# MeetingIQ – Intelligent Meeting Analysis System

MeetingIQ is an end-to-end meeting intelligence platform designed to transform raw multi-speaker meeting recordings into structured, explainable meeting insights. Built on the AMI Meeting Corpus, the system performs automatic speech transcription, speaker diarization, topic segmentation, topic modeling, action-item extraction, and explainable AI reporting.

---

# Features

## Speech Transcription

* Utilizes OpenAI Whisper for high-quality speech-to-text conversion.
* Supports long multi-speaker meeting recordings.
* Produces timestamped transcripts.

---

## Speaker Diarization

* Uses Pyannote Audio for speaker identification.
* Assigns speaker labels to individual utterances.
* Generates structured speaker-aware transcripts.

Example:

```text
Speaker A : Let's discuss the budget proposal.

Speaker B : We should complete it before Friday.

Speaker A : Agreed.
```

---

# Topic Segmentation

MeetingIQ implements multiple topic segmentation approaches for comparison and evaluation.

## TextTiling

* Lexical cohesion based segmentation algorithm.
* Computes depth scores to identify topic boundaries.
* Used to generate weak supervision labels for BiLSTM training.

---

## BiLSTM Topic Boundary Classifier

* SentenceTransformer embeddings are extracted for each utterance.
* Embeddings are passed through a bidirectional LSTM network.
* Predicts topic transition probabilities between utterances.
* Trained using TextTiling-generated boundary labels.

---

## LDA Topic Modeling

* Discovers latent discussion themes.
* Produces topic-word distributions.
* Used to analyze topical consistency within predicted segments.
* Benchmarked against TextTiling and BiLSTM outputs.

---

# Explainable AI Layer

MeetingIQ includes explainability modules to improve transparency of segmentation and action-item extraction.

---

## Boundary Visualization

Provides visual reports including

* TextTiling depth-score curves
* Topic coherence valleys
* Boundary confidence distributions

---

## BiLSTM Saliency Analysis

Gradient-based saliency maps are generated to identify utterances contributing most strongly to topic boundary predictions.

These visualizations help explain why the model selected specific segmentation points.

---

## Topic Interpretability

LDA outputs are visualized using

* Topic-word distributions
* Topic coherence scores
* Segment-level topic summaries

---

## Decision Traceability

Each generated report contains

* Predicted topic boundaries
* Topic assignments
* Speaker information
* Extracted action items
* Supporting transcript evidence

# Action Item Extraction

A hybrid extraction framework was developed using spaCy and FLAN-T5.

## spaCy

Used for

* Dependency parsing
* Part-of-speech tagging
* Imperative sentence detection
* Candidate task identification

---

## FLAN-T5

Used for

* Rewriting conversational statements
* Producing concise task descriptions
* Generating speaker-specific to-do items

# Evaluation Metrics

Topic segmentation methods are evaluated using the following metrics.

| Metric                | Description                              |
| --------------------- | ---------------------------------------- |
| Pk                    | Probability of segmentation disagreement |
| WindowDiff            | Window-based segmentation error          |
| Precision             | Correctly predicted boundaries           |
| Recall                | Coverage of true boundaries              |
| F1 Score              | Harmonic mean of precision and recall    |
| Segmentation Covering | Region overlap quality                   |
